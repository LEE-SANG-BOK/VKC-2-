# VKC Wizard Agent Persona (UI Guide)

본 문서는 “비자 사전평가 / 서류 자동화 / 상담 신청” 등 **Wizard 기반 플로우**에서 사용되는 **UI 가이드 페르소나**(카피/톤/행동 규칙)의 단일 소스(SoT)다.  
※ LLM/챗봇 구현이 아니라, **일관된 UX 카피/가이드 방식**을 고정하는 목적이다.

---

## 1) 페르소나 정의

- 이름(표기)
  - ko: **린(Linh)**, “K‑Connect 가이드”
  - vi: **Linh**, “Hướng dẫn K‑Connect”
- 역할
  - 사용자의 입력을 **단계적으로** 수집하고, 결과(확률/체크리스트/서류)를 이해하기 쉽게 정리하며, 다음 액션(서류 생성/상담 신청)으로 자연스럽게 연결한다.
- 성격/톤
  - 신뢰·명확·간결: 불필요한 수식/장문 금지
  - 친절하지만 과잉 감정표현 금지(과장·유행어·이모티콘 사용 금지)
  - “한국 생활/행정” 문맥을 전제로 한 단어 선택(항상 **한국**을 명시)

---

## 2) 언어/표현 규칙 (ko/vi)

- 원칙: **같은 의미**가 두 언어에서 동일하게 전달되도록 키를 동기화한다.
- 호칭
  - ko: 기본 존댓말(“해주세요/됩니다/가능해요”)
  - vi: 기본 “bạn” 사용(과도한 존칭 체계 강요 금지)
- 키워드 표기
  - 커뮤니티에서 통용되는 약어/코드는 그대로 사용: `TOPIK`, `EPS`, `D-2`, `D-10`, `E-7`, `F-2` 등
  - 다만 첫 등장(설명/도움말)에서는 풀어쓰기 1회 허용(예: “TOPIK(한국어능력시험)”)
- 문장 길이
  - 1문장 25자 내외(ko) / 90자 내외(vi)를 목표로 하되, “모바일 2줄”을 넘기지 않게 한다.
- 금지
  - 법률적 단정/보장 표현: “무조건 승인”, “100% 합격”, “정부가 보장” 등
  - 불필요한 경고/면책 장문(대신 **상담 CTA**로 처리)

---

## 3) Wizard 행동 규칙(UX)

### 3.1 진행 방식(기본)

- 한 화면에 “한 질문” 또는 “서로 강하게 연관된 2~3 필드”만 배치한다.
- 각 step은 다음 요소를 갖는다.
  - 질문(1줄)
  - 짧은 도움말(선택, 1~2줄)
  - 예시/포맷(placeholder)
  - 다음 버튼(하단 고정 CTA)
- 진행률은 항상 노출한다.
  - 예: `2 / 7` (Step indicator)

### 3.2 분기(Branching)

- 사용자 답변에 따라 “필요한 질문만” 묻는다(불필요 step 금지).
- 분기 로직은 상태머신(XState 등) 또는 단일 라우팅 규칙으로 관리하고, 화면별 임의 조건문 남발을 금지한다.

### 3.3 오류/검증 카피

- 원칙: **사용자 탓을 하지 않는다.**
- 오류는 “무엇을 어떻게” 고치면 되는지 한 줄로 제시한다.
  - ko 예: “여권 영문 이름과 동일하게 입력해주세요.”
  - vi 예: “Vui lòng nhập đúng theo tên tiếng Anh trên hộ chiếu.”

---

## 4) 결과 화면(확률/추천) 카피 규칙

- 결과는 한 화면에서 3단 구조로 고정한다.
  1) 요약(핵심 숫자/상태)
  2) 근거(점수/충족/미충족/부족 항목)
  3) 다음 액션(서류 생성/상담 신청)
- “확률(%)”은 시스템 정의(룰 기반 점수 매핑)로 계산되며, **비자 타입별 근거 항목**을 함께 보여준다.
- 상위 리드(예: ≥ 80%)는 상담 CTA를 1순위로 배치한다(상담 신청 → 관리자 리드 자동 생성 연동).

## 5) 공통 시스템 메시지 템플릿(예시)

### 5.1 시작(첫 진입)

- ko: “몇 가지 질문만 답하면 결과를 바로 보여드릴게요.”
- vi: “Chỉ cần trả lời vài câu hỏi, mình sẽ hiển thị kết quả ngay.”

### 5.2 일일 1회 제한(도달 시)

- ko: “오늘은 이미 사용했어요. 내일 다시 진행할 수 있어요.”
- vi: “Hôm nay bạn đã sử dụng rồi. Bạn có thể thử lại vào ngày mai.”

### 5.3 입력 누락(다음 단계 불가)

- ko: “이 항목을 입력하면 다음 단계로 넘어갈 수 있어요.”
- vi: “Vui lòng nhập mục này để tiếp tục.”
